<h1><a id="user-content-the-oceanpy-release-process" class="anchor" aria-hidden="true" href="#the-oceanpy-release-process"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>The ocean.py Release Process</h1>
<h2><a id="user-content-step-0-update-documentation" class="anchor" aria-hidden="true" href="#step-0-update-documentation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Step 0: Update documentation</h2>
<ul>
<li>Go to <a href="https://github.com/oceanprotocol/readthedocs">https://github.com/oceanprotocol/readthedocs</a>, and follow the steps</li>
<li>This will update what's shown in <a href="https://docs.oceanprotocol.com/references/read-the-docs/ocean-py/" rel="nofollow">https://docs.oceanprotocol.com/references/read-the-docs/ocean-py/</a>.</li>
</ul>
<p>This doesn't actually affect the pip release of the following steps. And if you've just updated READMEs, you can stop after this step if you like.</p>
<h2><a id="user-content-step-1-bump-version-and-push-changes" class="anchor" aria-hidden="true" href="#step-1-bump-version-and-push-changes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Step 1: Bump version and push changes</h2>
<ul>
<li>
<p>Create a new local feature branch, e.g. <code>git checkout -b feature/bumpversion-to-v0.2.5</code></p>
</li>
<li>
<p>Use the <code>bumpversion.sh</code> script to bump the project version. You can execute the script using {major|minor|patch} as first argument to bump the version accordingly:</p>
<ul>
<li>To bump the patch version: <code>./bumpversion.sh patch</code></li>
<li>To bump the minor version: <code>./bumpversion.sh minor</code></li>
<li>To bump the major version: <code>./bumpversion.sh major</code></li>
</ul>
</li>
<li>
<p>Commit the changes to the feature branch.</p>
<p><code>git commit -m "Bump version &lt;old_version&gt; -&gt; &lt;new_version&gt;"</code></p>
</li>
<li>
<p>Push the feature branch to GitHub.</p>
<p><code>git push origin feature/bumpversion-to-v0.2.5"</code></p>
</li>
</ul>
<h2><a id="user-content-step-2-merge-changes-to-main-branch" class="anchor" aria-hidden="true" href="#step-2-merge-changes-to-main-branch"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Step 2: Merge changes to main branch</h2>
<ul>
<li>
<p>Make a pull request from the just-pushed branch.</p>
</li>
<li>
<p>Wait for all the tests to pass!</p>
</li>
<li>
<p>Merge the pull request into the <code>main</code> branch.</p>
</li>
</ul>
<h2><a id="user-content-step-3-release" class="anchor" aria-hidden="true" href="#step-3-release"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Step 3: Release</h2>
<ul>
<li>
<p>To make a GitHub release (which creates a Git tag):</p>
<ul>
<li>Go to the ocean.py repo's Releases page <a href="https://github.com/oceanprotocol/ocean.py/releases">https://github.com/oceanprotocol/ocean.py/releases</a></li>
<li>Click "Draft a new release".</li>
<li>For tag version, put something like <code>v0.2.5</code></li>
<li>For release title, put the same value (like <code>v0.2.5</code>).</li>
<li>For the target, select the <code>main</code> branch, or the just-merged commit.</li>
<li>Describe the main changes. (In the future, these will come from the changelog.)</li>
<li>Click "Publish release".</li>
</ul>
</li>
</ul>
<h2><a id="user-content-step-4-verifiy" class="anchor" aria-hidden="true" href="#step-4-verifiy"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Step 4: Verifiy</h2>
